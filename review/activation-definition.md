# Our Fable Activation Definition

This is the working definition for the next 1–2 weeks.

## Activated family
A family is only considered activated when all 4 of these are true:

1. reserve completed
2. first invite sent
3. first contributor response received
4. parent sees the first meaningful item in the vault

If one of these has not happened, the family is not yet activated.

## Why this definition matters
It prevents false positives.

A reserve alone is not activation.
An invite alone is not activation.
A contribution that the parent never really feels is not activation.

The archive has to feel alive.

## Event checklist to measure
### Parent-side
- reserve_submitted
- onboarding_started
- first_invite_sent
- first_meaningful_item_seen

### Contributor-side
- invite_opened
- contributor_prompt_viewed
- contributor_response_started
- contributor_response_submitted
- contributor_confirmation_seen

### System outcome
- first_contributor_response_received
- first_family_activation_completed

## Suggested primary activation timestamp
Use the timestamp when the parent can see the first meaningful contributed item in the vault.

That is the moment the product stops being theoretical.

## What does not count as a meaningful item
- empty account setup
- child added with no content
- invite sent with no response
- internal system-generated placeholder content

## What does count
- a letter
- a voice note
- a photo with meaning
- a video
- a contributor response that is visibly attached to the child’s vault

## Working scorecard cuts
For each acquisition path, measure:
- reserve -> first invite sent
- first invite sent -> first contributor response
- first contributor response -> first meaningful item seen
- reserve -> full activation

By:
- source
- landing page
- audience
- creative angle

## Immediate operating rule
Do not scale traffic until this activation path is clear and repeatable.
